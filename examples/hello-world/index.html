Connection Diagnostics
?
Page served over HTTP/HTTPS
FieldTwin only sends postMessage to pages served over HTTP or HTTPS. If you opened this file directly as file://, it will never connect.

Fix: Run npx serve examples/hello-world and open http://localhost:3000 in FieldTwin.
?
Running inside a FieldTwin iFrame
FieldTwin only communicates with integrations embedded as iFrames. Opening this page in a standalone browser tab will not trigger loaded.

Fix: Add the URL as an integration in FieldTwin → Settings → Integrations → Add Integration.
?
Received loaded event from FieldTwin
The loaded event is sent by FieldTwin when your integration is opened. It contains the token, backendUrl, and subProjectId needed for everything else.
?
JWT token received
The token is required for all REST API calls. If it is missing, your integration may not have the correct permissions.
?
Backend URL received
The backendUrl is the base URL for all REST API calls. It must not have a trailing slash.
Example: https://backend.fieldtwin.com
?
SubProject ID received
The subProjectId is required for all project-scoped API calls. It comes from msg.subProject (not msg.subProjectId) in the loaded event.
?
API server is ready
APIServerIsReady must be true in the loaded event before calling the REST API. If it is false, wait for the apiPodIsReady event.
?
REST API call succeeds
A live GET request to /API/v1.10/project/-/subProject/{id}/stagedAssets using the received token and backendUrl.

Common failures:
• 401 — Token expired or wrong Authorization header format. Use Bearer {token}.
• 404 — Check backendUrl (no trailing slash) and subProjectId.
• CORS/Network error — The backendUrl may not allow requests from this origin. Check browser console.
